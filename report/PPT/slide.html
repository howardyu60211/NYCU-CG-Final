為何選擇 Deferred?

將 Geometry Complexity 與 Lighting Calculation 解耦 (Decouple)。

  • 效率 (Efficiency): 光照計算僅針對可見像素，不浪費效能。
  • 擴展性 (Scalability): 支援數百個動態光源而不顯著掉幀。

Core Rendering: Post-Processing

HDR Rendering

使用 GL_RGB16F 浮點數紋理允許亮度 > 1.0，保留高亮區域的細節，避免過曝時色彩斷層。

Bloom Effect

提取高於閾值的像素，並使用 Ping-Pong Buffers 進行 Gaussian Blur，創造柔和的光暈。

Gamma Correction

最終 Tone Mapping (Reinhard/ACES) 與 Gamma 2.2 校正，確保輸出到螢幕的顏色是正確且自然的。

Player Interface & Interaction

Control Features

  • Analog Input: 支援手把線性加速，提供細膩的油門控制。

  • Terrain Interaction: 車輛駛入草地時會因摩擦力改變而減速 (Grass Slowdown)。

  • Start Logic: 實作起跑燈倒數與 Jump Start Penalty (偷跑懲罰)，比照 F1 規則。

Environment: Screen Space Reflections
Ray-Marched Reflections

濕滑路面實作

  • 觸發條件 (Condition): 僅當材質的 Roughness < 0.1 (如水漬區域) 時觸發。
  • 演算法 (Algorithm): 在 View Space 使用 Ray-marching 追蹤 Depth Buffer。
  • 結果 (Result): 真實的水坑反射車輛與霓虹燈，增強沈浸感。

Environment: Atmosphere

Exponential Fog

體積霧 (Volumetric Fog)

模擬大氣密度對光線的散射。使用指數衰減公式 exp(-density * distance)，依據像素深度混合場景顏色，創造距離感與深度暗示。

降雨系統 (Rain System)

使用 GL_LINES 高效繪製雨滴，利用位置循環 (Wrap-around) 邏輯，在攝影機周圍創造無限降雨的錯覺。

Car Physics: Bicycle Model

Slip Angle Calculation

車輛動力學 (Vehicle Dynamics)

採用 Bicycle Model (單車模型)，將四輪簡化為前後兩輪 (Front/Rear) 進行模擬。

  • 轉向 (Steering): 計算輪胎的 Slip Angle (α)。
  • 側向力 (Lateral Force): F = C × α (過彎受力)。
  • 碰撞 (Collision): 在賽道 Mesh 上使用重心插值 (Barycentric Interpolation) 計算精確高度，防止穿模。

Challenge: The "Puddle" Problem

陷阱：全域一致性

情境： 雨轉晴的過渡。

若只對賽道材質做全域的 Roughness 調整：

  • Roughness = 0.8 → 瞬間全乾 (無積水)。

Engineering Challenges: Model Loading

Visual Glitches
UV 錯位 (Misalignment)
症狀：貼圖破碎、上下顛倒。原因：glTF (左上原點) 與 OpenGL (左下原點) 座標系不同。
v_texCoord = vec2(uv.x, 1.0 - uv.y);
邊緣雜訊 (Artifacts)
症狀：模型邊緣出現黑線。原因：`GL_REPEAT` 導致採樣到對面像素。
glTexParameteri(..., GL_CLAMP_TO_EDGE);
模型隱形 (Invisible)
症狀：車身透明。原因：頂點順序 (Winding Order) 導致錯誤的背面剔除 (Back-face Culling)。
glDisable(GL_CULL_FACE); // Hotfix
資料損壞 (Corruption)
症狀：紅/黃色雜訊。原因：`unsigned short` 資料被誤讀為 `float`。
val = (float)ushort_data / 65535.0f;
